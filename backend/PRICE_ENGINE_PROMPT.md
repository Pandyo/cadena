# CDA 코인 가격 변동 엔진 - 보안뉴스 감성 분석 프롬프트

## 📋 시스템 개요

보안뉴스 RSS 제목 분석을 통해 CDA 코인의 하루 가격 변동을 자동 계산하는 AI 기반 가격 엔진입니다.

- **데이터 소스**: 국내 보안뉴스(boannews) 하루 업데이트분만 사용
- **분석 단위**: RSS 제목 중심 감성 분석
- **변동 반영**: 실시간 업데이트 시마다 누적 적용

---

## 🎯 핵심 원칙

1. **제목 중심 분석**: 본문 전체보다 제목만 판별
2. **키워드 가중치**: 긍정/부정을 강함/약함으로 세분화
3. **합산 기반**: 단순 개수 비교가 아닌 가중치 합산
4. **실시간 누적**: 뉴스 업데이트될 때마다 변동률 즉시 적용
5. **보수적 변동폭**: 하루 최대 변동폭 제한으로 극단치 방지

---

## 📊 키워드 분류 체계

### ❌ 부정 키워드

**강함 (-2점 각각)**

- 해킹, 침해, 유출, 랜섬웨어, 악성코드
- 취약점, 제로데이, 계정탈취, 피싱
- 디도스, 변조, 백도어, 권한상승, 원격코드실행, 공급망공격

**약함 (-1점 각각)**

- 사고, 공격, 경고, 주의, 긴급
- 피해, 확산, 공개, 분석

### ✅ 긍정 키워드

**강함 (+2점 각각)**

- 패치, 차단, 탐지, 복구, 대응
- 강화, 개선, 보완, 암호화, 인증강화
- 점검, 격리

**약함 (+1점 각각)**

- 조치, 발표, 권고, 안내, 수정, 예방, 완료

## 🧮 점수 계산 규칙

### 1단계: 기사 단위 감성 점수

```
기사 점수 = 긍정 키워드 점수 합 - 부정 키워드 점수 합

판정:
  +2 이상      → 긍정 (상승 신호)
  -2 이하      → 부정 (하락 신호)
  -1 ~ +1      → 중립 (변동 없음)
```

**예시:**

- "해킹 패치 차단" → (-2) + (+2) + (+2) = +2 → **긍정**
- "유출 사고" → (-2) + (-1) = -3 → **부정**
- "취약점 공개" → (-2) + (-1) = -3 → **부정** (공개는 상황 악화)
- "공격 차단" → (-1) + (+2) = +1 → **중립**

### 2단계: 하루 단위 변동값 계산

```
뉴스량 점수 = (오늘 기사 수 - 기준 기사 수) / 5
감성 점수 = 하루 기사 점수 총합 / 기사 수
최종 변동값 = 0.7 × 뉴스량 점수 + 0.3 × 감성 점수
```

**설명:**

- **뉴스량 점수**: 평소보다 많은 뉴스 = 이슈 증가 신호 (가중치 70%)
- **감성 점수**: 기사들의 평균 감성 (가중치 30%)
- 기준 기사 수: 5개 (조정 가능)

### 3단계: 가격 변동 적용

```
신규가격 = 현재가격 × (1 + 변동값)

상한/하한 제한:
  하루 최대 상승: +5% (또는 +10%)
  하루 최대 하락: -5% (또는 -10%)

신규가격 = max(현재가격 × 0.9, min(현재가격 × 1.1, 계산된 가격))
```

## 🔄 실시간 업데이트 방식

### 기존 방식 (일일 배치)

```
자정 -> 00:15 UTC에 하루치 기사 모두 분석 -> 가격 한 번 업데이트
```

### 신규 방식 (실시간 누적)

```
1. 아침에 보안뉴스 기사 3개 → 점수 계산 → 가격 업데이트
2. 오후에 새 기사 2개 추가 → 누적 점수 재계산 → 가격 재업데이트
3. 저녁에 새 기사 1개 추가 → 누적 점수 재계산 → 가격 재업데이트

누적 로직:
- 오늘 날짜의 모든 기사 수집 (UTC 기준 또는 KST 기준)
- 매번 전체 점수 재계산 (이전 결과 누적 아님, 매번 fresh 계산)
- 변동값 최대 제한 유지
```

```javascript
    }
  ],
  source: String,                   // "rss_update" 또는 "manual"
  createdAt: Date,                  // 업데이트 시점
  updatedAt: Date
}
```

## 🛠️ 구현 사양

### 입력

- **RSS 피드**: 보안뉴스 (boannews) 오늘 분 기사만
- **기준 기사 수**: 5개
- **시간 기준**: KST (한국 표준시) 자정 ~ 익일 자정

### 처리

- 키워드 매칭 (정규표현식, 불가사례 포함)
- 중복 제거 (URL 기반 또는 제목 기반)
- 문맥 보정: 대응 키워드가 있어도 원인 키워드가 강하면 부정 우선

### 출력

- 신규 가격
- 변동률
- 기사별 감성 점수
- 뉴스량/감성 점수 분석

---

## 📝 운영 가이드

### 초기 설정

- 키워드 20개 내외로 시작
- 하루 변동폭 ±5% 또는 ±10% (조정 가능)
- 뉴스량/감성 가중치: 70%/30% (조정 가능)

### 유지보수

- 오탐이 많은 단어 제거
- 사건 중심 단어 우선순위
- 월 1회 이상 성능 검토

### 예외 처리

- RSS 피드 다운 시: 마지막 기사 시간 기준 유지
- 기사 0개: 중립 (변동 없음)
- 키워드 없는 기사: 중립

## 🚀 통합 흐름

```
1. 보안뉴스 RSS 구독 (실시간)
   ↓
2. 오늘 기사 필터링 (KST 자정 기준)
   ↓
3. 각 기사 제목 감성 분석
   ↓
4. 기사별 점수 계산
   ↓
5. 하루 누적 점수 계산
   ↓
6. 최종 변동값 도출
   ↓
7. 가격 업데이트 + DB 저장
   ↓
8. Frontend에 실시간 반영 (WebSocket 또는 polling)
```

## 📌 주요 차이점 (기존 vs 신규)

| 항목              | 기존                  | 신규                       |
| ----------------- | --------------------- | -------------------------- |
| **분석 대상**     | 해외+국내 RSS         | 국내 보안뉴스만            |
| **업데이트 주기** | 일일 배치 (00:15 UTC) | 실시간 누적                |
| **점수 방식**     | 기사 개수 기반        | 감성 분석 기반             |
| **조정 폭**       | 고정 ±2%~5%           | 동적 (최대 ±5%~10%)        |
| **저장 데이터**   | price, newsCount      | price, 감성점수, 기사 목록 |
